Sobre o Projeto

Este projeto foi desenvolvido como requisito para a avaliação da G2 na disciplina de desenvolvimento e análise de dados. O objetivo principal é transformar uma base de simulação robusta do mercado imobiliário nacional em um ecossistema analítico visual e inteligente.

A análise responde de maneira prática a problemas de negócio, mapeando a evolução temporal de preços indexados à taxa de juros e cruzando o poder aquisitivo (renda média regional) com os patamares de precificação de imóveis (Baixo, Médio, Alto e Luxo).

Tecnologias Utilizadas

A arquitetura da aplicação foi desenhada visando performance no carregamento de dados e isolamento de dependências, utilizando os pacotes mais sólidos do ecossistema Data Science:

Python Streamlit (Interface Web) Pandas (Manipulação) Seaborn & Matplotlib (Visualização) SQLAlchemy (ORM) SQLite (Banco de Dados Relacional)

Estrutura Organizacional

O repositório segue rigorosamente os padrões de organização profissional para projetos baseados em Python:

  • 📁 dados / — Contém a base bruta 'simulacao_mercado_imobiliario_brasil.csv'
  • 📄 app.py — Código principal da aplicação executiva Streamlit
  • 📄 notebook.ipynb — Notebook de exploração, análise de nulos e Engenharia de Atributos
  • 📄 requirements.txt — Lista de dependências e bibliotecas do projeto
  • 📄 mercado_imobiliario.sqlite — Banco de dados gerado dinamicamente via SQLAlchemy
  • 📄 index.html — Página de documentação e apresentação (GitHub Pages)

Perguntas Orientadoras

O projeto foi estruturado para responder às seguintes questões analíticas sobre o mercado imobiliário brasileiro:

  • 🏙️ Quais cidades possuem imóveis mais caros?
  • 📈 Houve valorização imobiliária ao longo do tempo?
  • 🗺️ Existem diferenças regionais relevantes?
  • 🏠 Quais tipos de imóveis possuem maior valor?
  • 💵 Existe relação entre renda e preço dos imóveis?
  • 🚀 Quais cidades apresentaram maior crescimento?
  • 🌡️ Há períodos de aquecimento ou retração do mercado?
